Instructivo para Configurar Envío de Correos desde DSpace con OAuth2

Introducción

Este instructivo describe cómo configurar el envío de correos en DSpace utilizando OAuth2 con Microsoft 365. Esto es necesario si tu organización ha habilitado el doble factor de autenticación (MFA).

Pasos a seguir

Paso 1: Configurar una Aplicación Empresarial en Microsoft Entra ID

  1. Accede al portal de administración de Microsoft Entra ID: https://entra.microsoft.com/.
  2. Selecciona tu aplicación empresarial (por ejemplo, DSpace SMTP Relay).
  3. En Permisos de API, asegúrate de agregar:
    • Mail.Send (Delegado).
    • User.Read (Delegado).
  4. Haz clic en Conceder consentimiento de administrador para estos permisos.

Paso 2: Generar un Secreto de Cliente

  1. Ve a la sección Certificados y secretos en la aplicación empresarial.
  2. Haz clic en Nuevo secreto de cliente.
    • Asigna un nombre (por ejemplo, DSpace SMTP).
    • Selecciona una duración (1 o 2 años).
  3. Copia el valor del secreto generado. Nota: No podrás verlo nuevamente.

Paso 3: Obtener los IDs de la Aplicación

  1. Ve a la sección Información general de la aplicación empresarial.
  2. Copia los siguientes valores:
    • ID de la aplicación (Client ID).
    • ID de directorio (Tenant ID).

Paso 4: Configurar el archivo dspace.cfg

Edita el archivo de configuración de DSpace (dspace.cfg) y agrega la siguiente configuración:

mail.server=smtp.office365.com
mail.server.port=587
mail.server.username=correoenvio@miempresa.com
mail.server.password=secreto_de_cliente
mail.server.auth=true
mail.server.useTLS=true
mail.server.useSSL=false
mail.server.from=correoenvio@miempresa.com

# Configuración de OAuth2
mail.oauth2.enabled=true
mail.oauth2.client_id=ID_de_la_aplicación
mail.oauth2.client_secret=secreto_de_cliente
mail.oauth2.tenant_id=ID_de_directorio
mail.oauth2.scope=https://graph.microsoft.com/.default
mail.oauth2.token_url=https://login.microsoftonline.com/ID_de_directorio/oauth2/v2.0/token
  • ID_de_la_aplicación: Obtenido en el Paso 3.
  • secreto_de_cliente: Generado en el Paso 2.
  • ID_de_directorio: Obtenido en el Paso 3.

Paso 5: Reiniciar DSpace

  1. Reinicia tu instancia de DSpace para aplicar los cambios:
    2. sudo systemctl restart tomcat

Paso 6: Probar el Envío de Correos

  1. Realiza una acción en DSpace que dispare un correo (por ejemplo, restablecimiento de contraseña).
  2. Si falla, revisa los registros en dspace.log para identificar errores.

Notas importantes

  • Asegúrate de que el puerto 587 esté abierto en el servidor y el firewall permita conexiones salientes a smtp.office365.com.
  • Si encuentras problemas, revisa las políticas de acceso condicional en Microsoft Entra ID para asegurarte de que no estén bloqueando el acceso SMTP.

¿Necesitas ayuda?

Si necesitas más ayuda, revisa los logs de DSpace o consulta con el administrador de Microsoft 365 para confirmar la configuración.
