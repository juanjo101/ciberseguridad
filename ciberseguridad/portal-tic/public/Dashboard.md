### 🚨 Brechas de Evidencia (Fase Operativa Pendiente)
*Requerimientos operativos indispensables para obtener los 100 puntos en el iTICge, exigidos por OGTIC y CNCS más allá de la fase documental.*

<div style="background: #fffbeb; border-left: 4px solid #f59e0b; padding: 15px; margin-bottom: 10px; border-radius: 4px;">
  <h4 style="margin: 0 0 5px 0; color: #b45309;">1. Formalización Institucional (Estructura)</h4>
  <p style="margin: 0; font-size: 0.9em; color: #78350f;"><strong>Requerimiento:</strong> Acta de conformación del Comité de Seguridad (CONTI) y Carta de Nombramiento del CISO.<br>
  <strong>Observación:</strong> La OGTIC rechazará el expediente si la Máxima Autoridad Ejecutiva (MAE) no ha formalizado el comité mediante firmas administrativas. <em>(Se recomienda generar estas plantillas inmediatamente)</em>.</p>
</div>

<div style="background: #fffbeb; border-left: 4px solid #f59e0b; padding: 15px; margin-bottom: 10px; border-radius: 4px;">
  <h4 style="margin: 0 0 5px 0; color: #b45309;">2. Gestión de Riesgos y Continuidad (BIA/DRP)</h4>
  <p style="margin: 0; font-size: 0.9em; color: #78350f;"><strong>Requerimiento:</strong> Matriz de Riesgos de TIC completada y Documento BIA con sistemas críticos.<br>
  <strong>Observación:</strong> Se requiere evidenciar los riesgos operativos reales de la BNPHU (ej. fallo de catálogo, hackeo web) y la firma de aceptación de riesgo residual por parte de la MAE.</p>
</div>

<div style="background: #fef2f2; border-left: 4px solid #ef4444; padding: 15px; margin-bottom: 10px; border-radius: 4px;">
  <h4 style="margin: 0 0 5px 0; color: #b91c1c;">3. Concientización del Personal</h4>
  <p style="margin: 0; font-size: 0.9em; color: #7f1d1d;"><strong>Requerimiento:</strong> Evidencias de charlas (fotos, listas de asistencia) y Acuerdos de Confidencialidad firmados.<br>
  <strong>Observación:</strong> Actualmente no hay registros operativos de inducciones de seguridad recientes impartidas al personal por parte de RRHH/TIC. Crítico para la auditoría física.</p>
</div>

<div style="background: #fef2f2; border-left: 4px solid #ef4444; padding: 15px; border-radius: 4px;">
  <h4 style="margin: 0 0 5px 0; color: #b91c1c;">4. Auditorías Técnicas y Seguridad Web</h4>
  <p style="margin: 0; font-size: 0.9em; color: #7f1d1d;"><strong>Requerimiento:</strong> Certificado SSL vigente y Reporte de Análisis de Vulnerabilidades (Escaneo técnico anual).<br>
  <strong>Observación:</strong> El Centro Nacional de Ciberseguridad (CNCS) monitorea de forma autónoma. Se penaliza directamente el iTICge si la infraestructura web gubernamental presenta vulnerabilidades abiertas o carece de HTTPS válido.</p>
</div>
